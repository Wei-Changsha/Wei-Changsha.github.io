Updated on 2020-03-25 系统调用 编译内核

在Ubuntu上增加系统调用，编译并启动内核

需求

在 Ubuntu 或其他 Linux 平台为 Linux 内核增加 1-3 个新的系统调用，并启用新的内核，编写一个应用程序测试新增加的系统调用是否能正确工作。

准备

1.获取Linux内核源代码 linux-4.15.3

官网：（下载速度慢）

https://www.kernel.org/

ftp：（下载速度快）

http://ftp.sjtu.edu.cn/sites/ftp.kernel.org/pub/linux/kernel/

2.解压Linux内核源代码

● sudo xz –d linux-4.15.3.tar.xz
● sudo tar –xvf linux-4.15.3.tar -C /usr/src/

步骤

1.添加函数声明：gedit ./include/linux/syscalls.h

2.修改内核代码：定义系统调用gedit ./kernel/sys.c

3.添加系统调用号：系统调用号一定不能和原来的一样

sudo gedit ./arch/x86/entry/syscalls/syscall_64.tbl

4.编译安装内核

  • 编译内核可能用到的依赖库安装（部分依赖未安装，在后续操作中根据提示安装）

    • sudo apt-get update
    • sudo apt-get install libncurses5-dev
    • sudo apt-get install build-essential openssl
    • sudo apt-get install flex
    • sudo apt-get install bison
    • sudo apt-get install openssl
    • sudo apt-get install libssl-dev

  • 配置内核编译参数

    • cd /usr/src/切换到刚才我们解压的内核源代码目录下
    • sudo make mrproper清除编译过程产生的中间文件，就是将内核代码还原为刚解压的状态，最好每次编译前执行一下。
    • sudo make menuconfig配置编译参数，编译的内核模块选择

  • 编译内核

    • sudo make bzImage 编译新的内核

  • 编译和安装modules

    • sudo make modules
    • sudo make modules_install

  • 安装内核 sudo make install

  • 启动新内核 配置/boot/grub/grub.conf

    • GRUB_TIME_OUT_STYLE=HIDDEN 这行前面加#号注释掉
    • GRUB_CMDLINE_LINUX_DEFAULT=”text” 引号内改为text

  • 进入启动菜单启动新内核（按shift键）

    • Ubuntu高级选项–>linux 4.15.3（Ubuntu启动正常时推荐使用）
    • Ubuntu高级选项–>linux 4.15.3(recovery mode)–>resume（如果Ubuntu启动时一直卡住，无法正常启动，则用recovery mode进入（我用的虚拟机，编译好内核驱动有问题））

  • 查看内核版本 在终端输入 uname -r

5.编写调用系统函数的c函数

6.测试分别依次输入：gcc hello.c -o hello 和 ./hello

发现打印出“System call sys_hello:0”，则运行成功

7.然后用命令dmesg查看结果：打印出“Hello！My student ID is xxxxxxxxx”，说明系统调用成功。
